# Security Policy / 安全策略

## Scope / 范围

This repository mainly stores agent skills, docs, demo assets, and small helper scripts.
Please report issues that could lead to:

- secret or token leakage
- unsafe or destructive command suggestions
- privacy or data exposure in examples, docs, or prompts
- script behavior that may damage user files, accounts, or local environments

这个仓库主要存放 agent skills、说明文档、效果图和少量辅助脚本。
请重点报告以下安全问题：

- 密钥、令牌或登录态泄露
- 会诱导执行危险或破坏性命令的技能行为
- 示例、文档或提示词里的隐私 / 数据暴露
- 可能破坏用户文件、账号或本地环境的脚本行为

## Reporting / 报告方式

- Do not post secrets, tokens, or exploit details in public issues.
- Prefer GitHub private vulnerability reporting if it is enabled for this repository.
- If private reporting is unavailable, open a minimal public issue and ask the maintainer for a private contact path.

- 不要在公开 issue 中提交密钥、令牌、登录态或完整漏洞利用细节。
- 如果仓库开启了 GitHub 私密漏洞报告，请优先使用该方式。
- 如果没有开启私密报告，请先提交一个最小公开 issue，并请求维护者提供私下沟通渠道。

## Response / 响应

This is a small repository, so response is best-effort.
The maintainer will try to acknowledge valid reports within 7 days.

这是一个较小的仓库，安全响应以 best-effort 为主。
维护者会尽量在 7 天内确认有效报告。

## Out of Scope / 不在范围内

- typos, wording issues, or general documentation polish
- feature requests
- broken examples that do not create a real security impact

- 纯文案问题或普通文档润色
- 功能建议
- 不构成真实安全影响的普通示例错误
